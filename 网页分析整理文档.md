# 绿脉·六驱全地形自适应抓取机器人网页分析整理文档

## 1. 项目概况

本网页是一个基于 Next.js 的静态展示站点，主题为“绿脉·六驱全地形自适应抓取机器人”。网页主要面向农业机器人项目展示，围绕六轮移动底盘、自主导航、视觉感知、路径规划、机械臂抓取和系统协同展开说明。

站点整体定位偏项目介绍与技术成果展示，适用于竞赛展示、项目汇报、技术演示入口或团队作品主页。

## 2. 技术栈与运行方式

项目采用的主要技术如下：

- 前端框架：Next.js 13.4.4，React 18.2.0
- 开发语言：TypeScript
- 样式方案：Tailwind CSS 3.3.2
- 动画库：AOS
- 弹窗组件：Headless UI Dialog 与 Transition
- 构建方式：Next.js 静态导出，`next.config.js` 中配置了 `output: 'export'`
- 图片优化：配置为 `images.unoptimized: true`，适合 GitHub Pages 等静态部署场景

常用命令：

```bash
npm install
npm run dev
npm run build
```

当前构建状态：已执行 `npm run build`，构建成功，生成静态页面 9 个。

## 3. 页面路由结构

项目使用 Next.js App Router，主要路由如下：

| 路由 | 页面说明 | 源文件 |
|---|---|---|
| `/` | 首页，项目核心展示页面 | `app/(default)/page.tsx` |
| `/about` | 关于我们，项目背景、基础与创新点 | `app/(default)/about/page.tsx` |
| `/learn-more` | 技术详情，核心技术与应用场景 | `app/(default)/learn-more/page.tsx` |
| `/signin` | 登录页，保留英文模板内容 | `app/(auth)/signin/page.tsx` |
| `/signup` | 注册页，保留英文模板内容 | `app/(auth)/signup/page.tsx` |
| `/reset-password` | 重置密码页，保留英文模板内容 | `app/(auth)/reset-password/page.tsx` |

默认展示类页面使用 `app/(default)/layout.tsx`，该布局引入了页面背景插图、AOS 动画初始化和页脚。

认证类页面使用 `app/(auth)/layout.tsx`，目前主要保留模板表单内容，未与机器人项目主题统一。

### 4.4 技术路线与整体系统设计

对应组件：`components/zigzag.tsx`

该区域展示两张核心图片：

- `public/images/技术路线.png`
- `public/images/整体系统设计.png`

第一部分“技术路线图”说明项目研发流程：

- 查阅研究背景和国内外发展现状
- 聚焦机器视觉、自主导航和智能决策
- 学习语义分割与路径规划
- 提出视觉感知算法和分层异构路径规划算法
- 仿真调试后进行机器人实机部署
- 引入 ROS2、MoveIt2 和机械臂子系统
- 将 YOLO 检测与投影链路接入机械臂规划脚本

第二部分“整体系统设计”说明机器人系统架构：

- 上位机通过树莓派与下位机通信
- 上位机负责图像处理、信息提取、数据处理和路径规划
- 下位机执行运动控制并反馈速度、转向角等状态
- MIU 惯导模块实时获取姿态信息
- 使用 PID 控制算法实现六轮机器人精确运动
- SO-101 机械臂关节与末端位姿纳入通信和时间同步框架
- 驻停阶段下发 MoveIt 轨迹，异常时可回退重定位

### 4.5 视觉感知算法设计

对应组件：`components/testimonials.tsx`

该区域标题为“看看我们的视觉感知算法设计”，核心方案是结合双分支融合网络 SFNet 与自监督学习的 RGB-D 视觉感知算法。

主要技术点：

- RGB 和深度图分别进入两个分支
- RGB 分支提取语义信息
- 深度分支提取几何与空间信息
- 通过特征互补模块进行多尺度融合
- 利用超像素作为参考，引导自监督训练
- 从深度图构建斜率、台阶高度、粗糙度等可通行性图
- 将分割图与可通行性图融合，生成最终通行区域

该区域包含 3 个卡片，每个卡片均可点击“了解更多”打开弹窗：

| 卡片 | 弹窗内容 |
|---|---|
| 自监督可通行性区域 | 自监督视觉预训练、表面法线估计、SLIC 超像素引导训练 |
| 网络架构 | SFNet 双分支融合、ResNet 主干、特征互补块、SwiftNet 风格上采样 |
| 可通行区域融合分割 | 多尺度特征融合、GateBlock、动态卷积核、条件卷积 |

### 4.6 智能路径规划算法

对应组件：`components/testimonials_2.tsx`

该区域标题为“还有我们的智能路径规划算法”，描述面向复杂 3D 场景的分层自主探索框架。

核心思路：

- 将路径规划划分为局部规划层和全局规划层
- 局部层维护高分辨率环境表示，生成动力学可行路径
- 全局层维护低分辨率表示，计算粗路径
- 通过随机采样选视点、构建路径、路径平滑和全局连接子空间完成探索
- 文案中提到探索效率提升 80%，计算量减少 50% 以上

该区域也包含 3 个可弹窗展开的技术卡片：

| 卡片 | 弹窗内容 |
|---|---|
| 视点采样 | 视点与表面点感知模型、最小覆盖视点采样、探索奖励驱动机制 |
| 局部路径规划 | 曲率约束、路径分段和平滑、TSP 顺序求解、贪心策略降低复杂度 |
| 全局规划 | 子空间状态建模、稀疏随机路线图、A 星搜索、TSP 遍历顺序优化 |

## 8. 当前实现状态与问题

### 8.1 已完成部分

- 首页核心展示链路完整
- 主页包含项目介绍、视频演示、难题拆解、技术路线、视觉算法、路径规划算法
- `/about` 与 `/learn-more` 已完成中文内容
- 静态构建通过，可用于静态部署
- 主要图片、视频和 PDF 资源已放入 `public` 目录

### 8.2 需要关注的问题

1. 根布局元数据仍是模板内容  
   `app/layout.tsx` 中 metadata 仍为 `Create Next App` 和 `Generated by create next app`，建议改为项目名称和正式描述。

2. 页眉组件未被使用  
   `components/ui/header.tsx` 存在，但未在布局中引入。当前页面顶部没有统一导航栏，访问 `/about` 或 `/learn-more` 后主要依赖浏览器返回或页脚链接回到首页。

3. 登录、注册、重置密码页面仍是英文模板  
   `/signin`、`/signup`、`/reset-password` 保留 Open PRO 模板内容，与农业机器人项目无关。如果不需要认证功能，建议移除或隐藏这些路由。

4. 部分文案仍有模板残留  
   `components/zigzag.tsx` 中出现 “More speed. Less spend”，与当前项目主题不匹配。

5. 未使用组件较多  
   `components/blocks.tsx`、`newsletter.tsx`、`banner.tsx` 等组件未在首页使用，部分仍是英文模板内容。

6. 弹窗内容存在 HTML 结构问题  
   `components/testimonials_2.tsx` 中局部路径规划弹窗存在嵌套 `<p>` 标签和重复段落，虽然当前构建通过，但不利于语义结构和后续维护。

7. 项目命名不统一  
   README 中描述为“面向复杂地形的运输机器人自主感知导航系统”，页面中描述为“六驱全地形自适应抓取机器人”或“采摘机器人”，建议统一项目名称和关键词。

8. 浏览器兼容数据库提示过期  
   构建时提示 `caniuse-lite is outdated`，可按需更新 Browserslist 数据库。

## 9. 优化建议

建议按以下优先级优化：

1. 统一站点标题、描述、项目名称和关键词。
2. 将 `Header` 引入默认布局，增加首页、关于我们、技术详情、技术文档等导航入口。
3. 移除或改造认证模板页面，避免无关英文内容影响项目专业性。
4. 清理未使用模板组件，减少维护成本。
5. 修复 `testimonials_2.tsx` 中重复段落和嵌套标签问题。
6. 将技术路线、视觉算法、路径规划算法文案进一步压缩，页面展示保留重点，详细内容放入 PDF 或弹窗。
7. 为图片添加更准确的 `alt` 文案，提升可访问性和搜索友好性。
8. 将项目介绍中的技术指标单独整理为“成果与验证”模块，使竞赛展示更有说服力。

## 10. 总结

该网页已经具备完整的项目展示骨架，内容覆盖项目背景、机器人系统结构、核心技术路线、视觉感知算法、路径规划算法和实机演示视频。整体适合作为农业机器人竞赛或项目展示页面。

当前主要短板不是功能缺失，而是模板残留、导航缺失、命名不统一和局部文案结构不够精炼。完成这些整理后，网页会更像一个成熟的项目展示站点，也更适合用于项目展示和对外传播。
